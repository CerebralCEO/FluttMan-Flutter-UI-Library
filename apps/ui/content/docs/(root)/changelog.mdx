## Apr 25, 2026

The FlutMan documentation has been completely rewritten to reflect the Flutter ecosystem:

- **Introduction** — Now describes FlutMan as a Flutter UI component library
- **Get Started** — Features Flutter CLI workflow: `dart pub global activate flutman_cli`
- **Styling** — Rewritten for Flutter ThemeData and Material Design 3
- **Skills** — Updated for Copy Prompt button workflow

## Apr 17, 2026

### Form Component Update

The **`FlutManForm`** component no longer applies default layout padding. It is now a thin wrapper that you configure with standard Flutter layout widgets.

**Migration:**

For a typical stacked-field form, wrap with Column:

```dart
// Before
FlutManForm(
  child: Column(...),
)

// After - Use Column with padding
Padding(
  padding: EdgeInsets.all(16),
  child: Column(
    crossAxisAlignment: CrossAxisAlignment.stretch,
    children: [...],
  ),
)
```

For **dialogs and sheets**, structure your layout:

```dart
FlutManDialog(
  header: FlutManDialogHeader(...), // Outside the form
  content: FlutManForm(
    child: Column(
      children: [
        FlutManDialogPanel(...),
        FlutManDialogFooter(...),
      ],
    ),
  ),
)
```

## Apr 14, 2026

### OTP Field Component

The OTP Field component has been updated with a new API based on Flutter's native input handling.

**Migration:**

```diff
- flutman add input-otp
+ flutman add otp-field
```

```dart
// New API
FlutManOTPField(
  length: 6,
  onCompleted: (code) => print(code),
)
```

See the [OTP Field](/ui/docs/components/otp-field) docs for full API details.

## Apr 12, 2026

### Table Component

`FlutManTable` now supports an optional **`variant`** parameter: **`TableVariant.default`** (the default) or **`TableVariant.card`**. Use **`TableVariant.card`** for the card-style grid with separated rows and rounded cells.

**Migration:**

No change required for existing tables. Opt in where you want the card-style table:

```dart
FlutManTable(
  variant: TableVariant.card,
  children: [...],
)
```

## Mar 20, 2026

### Toggle Group

`FlutManToggle` was renamed to `FlutManToggleGroupItem` in the toggle group to avoid naming conflicts with the standalone toggle widget.

**Migration:**

```diff
- import 'package:my_app/ui/components/toggle.dart' show FlutManToggle, FlutManToggleGroup;
+ import 'package:my_app/ui/components/toggle_group.dart' show FlutManToggleGroup, FlutManToggleGroupItem;
```

```diff
  FlutManToggleGroup(
    children: [
-     FlutManToggle(value: "bold", child: Text("Bold")),
+     FlutManToggleGroupItem(value: "bold", child: Text("Bold")),
    ],
  )
```
