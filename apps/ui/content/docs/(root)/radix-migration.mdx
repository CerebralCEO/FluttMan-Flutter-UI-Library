This guide is for Flutter developers who want to adopt **FlutMan** in their existing applications.

**FlutMan** is a zero-dependency, copy-paste UI component library for Flutter. Unlike traditional packages, FlutMan components live directly in your project's source code, giving you full control and customization.

## Overview

Migrating to FlutMan from raw Flutter widgets or other UI libraries is straightforward. Since FlutMan components are just Dart files in your project, you can:

- **Migrate gradually** â€” Replace widgets one at a time
- **Customize freely** â€” Edit any component to match your needs
- **Keep your existing code** â€” No breaking changes to your app structure

This guide provides patterns for common migration scenarios.

## From Raw Flutter Widgets

Many Flutter apps use built-in Material or Cupertino widgets directly. Here's how to migrate common patterns to FlutMan components.

### Buttons

**Before (Material Button):**

```dart
ElevatedButton(
  onPressed: () {},
  style: ElevatedButton.styleFrom(
    backgroundColor: Theme.of(context).colorScheme.primary,
    foregroundColor: Theme.of(context).colorScheme.onPrimary,
    padding: EdgeInsets.symmetric(horizontal: 16, vertical: 12),
    shape: RoundedRectangleBorder(
      borderRadius: BorderRadius.circular(8),
    ),
  ),
  child: Text('Click me'),
)
```

**After (FlutManButton):**

```dart
FlutManButton(
  onPressed: () {},
  variant: ButtonVariant.filled,
  child: Text('Click me'),
)
```

### Cards

**Before (Material Card):**

```dart
Card(
  elevation: 2,
  shape: RoundedRectangleBorder(
    borderRadius: BorderRadius.circular(12),
  ),
  child: Padding(
    padding: EdgeInsets.all(16),
    child: Column(
      crossAxisAlignment: CrossAxisAlignment.start,
      children: [
        Text('Title', style: Theme.of(context).textTheme.titleLarge),
        SizedBox(height: 8),
        Text('Content here...'),
      ],
    ),
  ),
)
```

**After (FlutManCard):**

```dart
FlutManCard(
  child: FlutManCardHeader(
    title: Text('Title'),
    description: Text('Content here...'),
  ),
)
```

### Dialogs

**Before (Material Dialog):**

```dart
showDialog(
  context: context,
  builder: (context) => AlertDialog(
    title: Text('Confirm'),
    content: Text('Are you sure?'),
    actions: [
      TextButton(
        onPressed: () => Navigator.pop(context),
        child: Text('Cancel'),
      ),
      TextButton(
        onPressed: () => Navigator.pop(context),
        child: Text('Confirm'),
      ),
    ],
  ),
);
```

**After (FlutManDialog):**

```dart
FlutManDialog(
  trigger: FlutManButton(
    child: Text('Open Dialog'),
  ),
  header: FlutManDialogHeader(
    title: Text('Confirm'),
    description: Text('Are you sure?'),
  ),
  footer: FlutManDialogFooter(
    actions: [
      FlutManButton(
        variant: ButtonVariant.ghost,
        onPressed: () => Navigator.pop(context),
        child: Text('Cancel'),
      ),
      FlutManButton(
        onPressed: () => Navigator.pop(context),
        child: Text('Confirm'),
      ),
    ],
  ),
)
```

## From Other Flutter UI Libraries

If you're using another Flutter UI library, migration follows a similar pattern.

### Step 1: Install FlutMan

Activate the CLI and initialize your project:

```bash
dart pub global activate flutman_cli
flutman init
```

### Step 2: Add Equivalent Components

Find the FlutMan equivalent of your current components:

| Common Library | FlutMan Equivalent |
| -------------- | ------------------ |
| `flutter_material_color_utilities` | Built into FlutMan theming |
| Custom button widgets | `FlutManButton` |
| `flutter_staggered_grid_view` cards | `FlutManCard`, `FlutManCardFrame` |
| Custom dialog implementations | `FlutManDialog`, `FlutManSheet` |
| `flutter_slidable` | `FlutManListItem` with actions |

### Step 3: Replace Gradually

Replace components one screen at a time:

1. Add the FlutMan component using `flutman add <component>`
2. Replace the import statement
3. Update the widget usage
4. Test the screen
5. Commit your changes

## Component Migration Guides

### Form Fields

**Before (TextFormField):**

```dart
TextFormField(
  decoration: InputDecoration(
    labelText: 'Email',
    hintText: 'Enter your email',
    border: OutlineInputBorder(),
    errorText: emailError,
  ),
  validator: (value) {
    if (value?.isEmpty ?? true) return 'Email is required';
    return null;
  },
)
```

**After (FlutManInput):**

```dart
FlutManField(
  label: Text('Email'),
  input: FlutManInput(
    hintText: 'Enter your email',
  ),
  error: emailError != null ? Text(emailError!) : null,
)
```

### Lists

**Before (ListView with ListTile):**

```dart
ListView.builder(
  itemCount: items.length,
  itemBuilder: (context, index) {
    return ListTile(
      leading: Icon(Icons.person),
      title: Text(items[index].name),
      subtitle: Text(items[index].email),
      trailing: Icon(Icons.chevron_right),
      onTap: () {},
    );
  },
)
```

**After (FlutManList):**

```dart
FlutManListView.builder(
  itemCount: items.length,
  itemBuilder: (context, index) {
    return FlutManListItem(
      leading: FlutManAvatar(
        child: Icon(Icons.person),
      ),
      title: Text(items[index].name),
      subtitle: Text(items[index].email),
      trailing: Icon(Icons.chevron_right),
      onTap: () {},
    );
  },
)
```

### Navigation

**Before (BottomNavigationBar):**

```dart
Scaffold(
  bottomNavigationBar: BottomNavigationBar(
    currentIndex: _selectedIndex,
    onTap: (index) => setState(() => _selectedIndex = index),
    items: [
      BottomNavigationBarItem(icon: Icon(Icons.home), label: 'Home'),
      BottomNavigationBarItem(icon: Icon(Icons.search), label: 'Search'),
      BottomNavigationBarItem(icon: Icon(Icons.settings), label: 'Settings'),
    ],
  ),
)
```

**After (FlutManBottomNav):**

```dart
FlutManBottomNavigation(
  currentIndex: _selectedIndex,
  onTap: (index) => setState(() => _selectedIndex = index),
  items: [
    FlutManBottomNavItem(icon: Icons.home, label: 'Home'),
    FlutManBottomNavItem(icon: Icons.search, label: 'Search'),
    FlutManBottomNavItem(icon: Icons.settings, label: 'Settings'),
  ],
)
```

## Theme Migration

When migrating, you'll want to preserve your app's visual identity.

### Preserving Colors

Map your existing colors to FlutMan's theme system:

```dart
// Before
MaterialApp(
  theme: ThemeData(
    primaryColor: Color(0xFF1E40AF),
    colorScheme: ColorScheme.light(
      primary: Color(0xFF1E40AF),
      secondary: Color(0xFF3B82F6),
    ),
  ),
)

// After - FlutMan respects your existing ThemeData
MaterialApp(
  theme: ThemeData(
    colorScheme: ColorScheme.fromSeed(
      seedColor: Color(0xFF1E40AF),
    ),
  ),
)
```

### Customizing FlutMan Components

Since you own the source code, customize components directly:

```dart
// Edit lib/ui/components/button.dart
class FlutManButton extends StatelessWidget {
  // Modify default values
  static const defaultPadding = EdgeInsets.symmetric(
    horizontal: 20, // Your custom padding
    vertical: 12,
  );
  
  @override
  Widget build(BuildContext context) {
    // Your customized implementation
  }
}
```

## Verify After Migration

After converting components, verify your app:

- **Visual consistency** â€” Check that colors, spacing, and typography match your design
- **Interaction patterns** â€” Test tap targets, gestures, and animations
- **Accessibility** â€” Verify screen reader support and focus management
- **Performance** â€” Ensure no regressions in scroll or animation performance
- **State management** â€” Confirm form validation and state handling work correctly

## Common Pitfalls

### Import Path Updates

Remember to update imports throughout your app:

```dart
// Before
import 'package:flutter/material.dart';

// After - Add FlutMan imports
import 'package:flutter/material.dart';
import 'package:my_app/ui/components/button.dart';
import 'package:my_app/ui/components/card.dart';
```

### Widget Namespacing

FlutMan components are prefixed to avoid conflicts:

| Before | After |
|--------|-------|
| `Button` | `FlutManButton` |
| `Card` | `FlutManCard` |
| `Input` | `FlutManInput` |
| `Dialog` | `FlutManDialog` |

### State Handling

Some components handle state differently:

- **Forms**: FlutMan forms use Flutter's built-in form validation
- **Dialogs**: Return values through `Navigator.pop(context, result)`
- **Lists**: Support both builder patterns and static children

Use this checklist for systematic migration:

- [ ] Install FlutMan CLI: `dart pub global activate flutman_cli`
- [ ] Initialize project: `flutman init`
- [ ] Identify components to migrate
- [ ] Add FlutMan components: `flutman add <component>`
- [ ] Update imports in target files
- [ ] Replace widget usage
- [ ] Test interactions and accessibility
- [ ] Update theme if needed
- [ ] Document customizations for your team
