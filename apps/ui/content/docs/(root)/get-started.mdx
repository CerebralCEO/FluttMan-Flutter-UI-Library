This guide provides the essentials for adding **FlutMan** components to your Flutter application.

## Prerequisites

Before you begin, make sure you have:

- **Flutter SDK** installed (version 3.0 or higher)
- **Dart SDK** (comes with Flutter)
- An existing Flutter project or create a new one with `flutter create my_app`

## Installation

FlutMan uses a CLI-based workflow to download components directly into your project. No packages to add to `pubspec.yaml`—just pure Dart code you own.

### Step 1: Activate the CLI

Install the FlutMan CLI globally using Dart's pub command:

```bash
dart pub global activate flutman_cli
```

This makes the `flutman` command available anywhere on your system.

### Step 2: Initialize Your Project

Navigate to your Flutter project root and initialize FlutMan:

```bash
flutman init
```

This command:
- Creates the `lib/ui/components/` directory structure
- Sets up the base theme configuration
- Prepares your project for FlutMan components

### Step 3: Add a Component

Now you can add any component to your project. For example, to add a Button:

```bash
flutman add button
```

The CLI will:
- Download the Button component Dart file to `lib/ui/components/`
- Install any required dependencies to your `pubspec.yaml`
- Run `flutter pub get` automatically

## Adding Components

### With the CLI (Recommended)

Each component page provides a command to add it to your project automatically. The CLI handles everything—file creation, dependency management, and project setup.

```bash
flutman add <component-name>
```

### Available Commands

- `flutman add <component>` — Add a specific component
- `flutman list` — See all available components
- `flutman init` — Initialize or re-initialize FlutMan in a project

### Manually

If you prefer, you can also manually copy components:

1.  **Find a component** on the [Components](/ui/docs) page.
2.  **Copy the code** from the **Code** tab.
3.  **Create a new file** in your project at `lib/ui/components/<component>.dart`.
4.  **Paste the code** and update any import paths as needed.
5.  **Add dependencies** if the component requires external packages.

## Using Components

Once added, import and use components in your Flutter widgets:

```dart
import 'package:my_app/ui/components/button.dart';

class MyScreen extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Scaffold(
      body: Center(
        child: FlutManButton(
          onPressed: () {},
          child: Text('Get Started'),
        ),
      ),
    );
  }
}
```

## Styling

FlutMan components use Flutter's built-in theming system. Each component accepts standard Flutter styling properties and can be customized to match your design system.

Components are designed to work with:
- **Material Design 3** (default Flutter theme)
- **Custom themes** via `ThemeData`
- **Cupertino** styling for iOS-style apps

See the [Styling](/ui/docs/styling) guide for more details on customizing the look and feel.

## Working with LLMs

We structure the documentation to make the components **AI-friendly**, so language models can understand, reason about, and modify them. To support this, we include:

- **[Agent Skills](/ui/docs/skills)** — Install FlutMan knowledge directly into your AI assistant.
- A **Copy Prompt** button on every component page that generates AI-ready instructions.
